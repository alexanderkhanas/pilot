Lemonn Documentation
NOTE: Every call but `/user/signup` and `/user/login` needs to send the JWT via the `Authorization` header.

/ – GET
Redirects to Lemonn home page
Params: none

/participant/signup – POST
Sign up participant
Params: `email`, `password`

/participant/signup/verify/:token – GET
Participant signup verify
Params: `token` (as URL slug)

/participant/login – POST
Log in participant
Params: `email`, `password`

/helpdesk/participants – GET
Returns the list of participants that need help
Params: none

/room/join – POST
Join a room
Params: `conferenceId`, `roomId`, `roomType`

/host/conference/create – POST
Create conference
Params: `eventTitle`, `eventStartsAt`, `maxParticipants`, `maxManagersWaitingRooms`, `maxParticipantsWaitingRooms`, `eventSchedule`

/conference/next – GET
Next conference available
Params: none

/conference/rooms – POST
List available rooms
Params: `conferenceId`

/host/participants/import – POST
Import participants CSV
Params: `csvFile`

/conference/:id/helpdesk – POST
Creates an HelpdeskTicket and a helpdesk room linked to the provided Conference by id.
Params: `conferenceId`
